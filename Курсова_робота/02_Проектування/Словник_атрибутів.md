## Примітки

- Усі таблиці використовують GUID (Globally Unique Identifier) для первинних ключів для забезпечення унікальності в розподіленій системі
- Типи даних відповідають SQL Server синтаксису
- ENUM типи представлені для зручності, у реальній реалізації можуть бути замінені на VARCHAR з CHECK constraints або окремі довідкові таблиці
- INDEX вказано для полів, які часто використовуються у WHERE та JOIN операціях
- DEFAULT значення встановлені для автоматичного заповнення полів
- CHECK constraints забезпечують валідацію даних на рівні БД
- Всі FOREIGN KEY зв'язки забезпечують цілісність даних
