# Wireframes (Каркаси інтерфейсу користувача)

## Система: SafeHeight Monitor

Wireframes (каркаси) - це структурна, але приблизна візуалізація інтерфейсу користувача, яка показує основні елементи та потік взаємодії без деталізації зовнішнього вигляду.

---

## Список Wireframes

### WF-01: Екран автентифікації та входу в систему
**Файл:** `WF-01_login_authentication.puml`

**Опис:** Головний екран входу в систему з підтримкою множинних ролей користувачів

**Основні елементи:**
- Вибір ролі користувача (5 ролей: Працівник, Інженер з ОП, Бригадир, Менеджер, Адміністратор)
- Три методи автентифікації:
  - QR-код сканування (для швидкого входу працівників)
  - PIN-код (6 цифр)
  - Логін/Пароль (для адміністративних ролей)
- Функція відновлення пароля
- Посилання на реєстрацію нового користувача
- Індикація безпечного з'єднання
- Контактна інформація підтримки

**Користувачі:** Всі ролі системи

**NFR:** NFR1.1, NFR1.5, NFR6.1, NFR6.2

---

### WF-02: Панель моніторингу в реальному часі
**Файл:** `WF-02_monitoring_realtime_dashboard.puml`

**Опис:** Головна робоча панель для інженера з охорони праці для моніторингу всіх працівників на будівельному майданчику

**Основні елементи:**
- Верхня навігаційна панель з вкладками (Моніторинг, Звіти, Налаштування, IoT)
- Панель управління з функціями:
  - Автооновлення даних (кожні 5 секунд)
  - Пошук працівників
  - Фільтри за статусом та зоною
  - Сортування за рівнем ризику
  - Експорт даних
- Таблиця працівників з деталями:
  - ID працівника
  - ПІБ
  - Статус (Критично/Увага/Безпечно) з кольоровою індикацією
  - Локація (зона та поверх)
  - Рівень ризику у відсотках
  - Час останнього оновлення
  - Дії (Деталі, Зв'язок)
- Інтерактивна карта будівельного майданчику:
  - Візуалізація зон
  - Позиціювання працівників (кольорові маркери)
  - Можливість повноекранного режиму
  - Доступ до налаштування зон
- Статус-бар внизу з агрегованою інформацією:
  - Загальна кількість працівників
  - Розподіл за статусами
  - Офлайн користувачі
  - Час останнього оновлення
  - Статус з'єднання

**Користувачі:** Інженер з охорони праці, Бригадир, Менеджер проекту

**NFR:** NFR2.1, NFR2.2, NFR2.3, NFR2.4, NFR3.1

### WF-03: Екран попередження на IoT пристрої працівника
**Файл:** `WF-03_worker_iot_alert.puml`

**Опис:** Мобільний інтерфейс критичного попередження про виявлений ризик падіння на IoT браслеті/смарт-касці

**Основні елементи:**
- Status bar мобільного пристрою (час, WiFi, Bluetooth, батарея)
- Великий заголовок попередження (червоний колір, великий шрифт)
- Панель інформації про загрозу:
  - Рівень небезпеки (Критичний/Увага)
  - Ймовірність інциденту у відсотках
  - Час виявлення
- Детальний список виявлених загроз:
  - Небезпечна висота (значення, критичний поріг)
  - Нестабільне положення тіла (кут нахилу)
  - Близькість до краю (відстань)
  - Відсутність страхувального троса
- Рекомендації з безпеки (нумерований список дій)
- Додаткова інформація:
  - Пульс працівника
  - Точна локація (зона, поверх)
- Дві великі кнопки для реакції:
  - "Я в безпеці" (зелена) - підтвердження
  - "Потрібна допомога!" (червона) - аварійний виклик
- Статус автоматичних сповіщень:
  - Інженер з охорони праці
  - Бригадир
  - Центральна диспетчерська
- Інформація про пристрій (ID, з'єднання, GPS)

**Користувачі:** Працівник будівельного майданчику

**NFR:** NFR3.1, NFR3.2, NFR3.3, NFR7.1, NFR7.2

### WF-04: Екран керування робочими зонами
**Файл:** `WF-04_zones_management.puml`

**Опис:** Інтерфейс для бригадира для створення, редагування та керування робочими зонами і призначення працівників

**Основні елементи:**
- Навігаційна панель з розділами (Панель, Зони, Бригада, Обладнання, Звіти)
- Панель інструментів для роботи з зонами:
  - Створити нову зону
  - Редагувати зону
  - Видалити зону
  - Експорт списку
  - Перемикач виду (Карта + Таблиця / Тільки карта / Тільки таблиця)
- Інтерактивна карта майданчику з інструментами:
  - Малювання нової зони
  - Переміщення зон
  - Зум (збільшення/зменшення)
  - Повноекранний режим
- Візуалізація зон на карті:
  - Назва зони та тип робіт
  - Діапазон поверхів
  - Маркери працівників (кольорові точки)
  - Індикація статусу зони (OK/Увага)
- Таблиця існуючих зон:
  - ID зони
  - Назва та опис
  - Поверхи
  - Площа
  - Кількість працівників (поточна/максимальна)
  - Тип робіт
  - Статус зони
  - Час останнього оновлення
  - Дії (Деталі, Редагувати, Видалити)
- Панель "Моя бригада":
  - Таблиця працівників з деталями:
    - ID, ПІБ
    - Призначена зона
    - Статус безпеки
    - Кваліфікація (розряд)
    - Час роботи на зміні
    - Дії (Переназначити, Зв'язок)
  - Функції управління:
    - Додати працівника
    - Масове переназначення
    - Експорт розкладу

**Користувачі:** Бригадир, Інженер з охорони праці

**NFR:** NFR4.1, NFR4.2, NFR4.3, NFR9.1

### WF-05: Дашборд аналітики та звітності
**Файл:** `WF-05_analytics_reporting.puml`

**Опис:** Аналітична панель для менеджера проекту з детальними звітами, трендами та рекомендаціями з безпеки

**Основні елементи:**
- Верхня навігація (Панель, Звіти, Документи, Налаштування)
- Панель конфігурації звіту:
  - Вибір періоду (попередні налаштування або кастомний діапазон дат)
  - Генерація звіту
  - Експорт у різних форматах (PDF, Excel)
  - Відправка email
  - Друк
- Ключові показники (KPI):
  - Всього попереджень за період
  - Розподіл за рівнями (Увага/Критично) у відсотках
  - Середня кількість на день
  - Пікове значення (дата)
  - Точність AI моделі
  - Середній час реакції системи
  - Кількість реальних інцидентів
- Графік трендів попереджень:
  - Візуалізація динаміки за період
  - Розділення за рівнями (Увага/Критично)
  - Перемикач періоду (День/Тиждень/Місяць)
  - Функції зуму та завантаження даних
- Таблиця "Топ-5 зон з найбільшим ризиком":
  - Ранг
  - Назва зони та тип робіт
  - Кількість попереджень
  - Відсоток від загальної кількості
  - Тренд (зростання/зниження у відсотках)
  - Дії (Деталі, Рекомендації)
- Аналіз виявлених ризиків:
  - Типи ризиків з кількістю та відсотками
  - Візуальна індикація (progress bars)
- Метрики ефективності системи:
  - Точність AI моделі (з метою)
  - Відсоток хибних спрацювань
  - Середній час реакції
  - Uptime системи
  - Візуалізація досягнення мети (progress bars)
- Панель автоматичних рекомендацій:
  - Пріоритизація (Високий/Середній/Низький)
  - Детальний опис проблеми
  - Конкретні рекомендації щодо усунення
- Футер звіту:
  - Час генерації
  - Автор звіту
  - Період
  - Статус (Затверджено/Чернетка)
  - Функції шерингу (Копіювати посилання, Зберегти як шаблон, Поділитися)

**Користувачі:** Менеджер проекту, Інженер з охорони праці, Системний адміністратор

**NFR:** NFR8.1, NFR8.2, NFR8.3, NFR10.1, NFR10.2

## Характеристики Wireframes

### Основні принципи дизайну:

1. **Структурна чіткість**
   - Чітка візуальна ієрархія елементів
   - Логічне групування функціоналу
   - Послідовний потік користувацьких дій

2. **Функціональна повнота**
   - Всі необхідні елементи інтерфейсу присутні
   - Показані основні потоки взаємодії
   - Враховані різні стани системи (нормальний, попередження, критичний)

3. **Кольорова індикація стану**
   - <color:green>Зелений</color> - безпечний стан, успішна операція
   - <color:orange>Оранжевий</color> - попередження, увага потрібна
   - <color:red>Червоний</color> - критична ситуація, негайна дія потрібна
   - Сірий - неактивний стан, допоміжна інформація

4. **Адаптивність**
   - Desktop інтерфейси (WF-01, WF-02, WF-04, WF-05)
   - Mobile інтерфейс (WF-03)
   - Responsive дизайн для різних розмірів екранів

5. **Доступність (Accessibility)**
   - Великі кнопки для критичних дій
   - Чіткі мітки та підказки
   - Контрастні кольори для важливої інформації
   - Іконки + текст для кращого розуміння
