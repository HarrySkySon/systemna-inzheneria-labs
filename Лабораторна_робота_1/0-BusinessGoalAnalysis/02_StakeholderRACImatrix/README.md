# RACI матриця для проекту моніторингу будівельних майданчиків

## Що таке RACI?

- **R (Responsible)** - Відповідальний за виконання
- **A (Accountable)** - Підзвітний, затверджує результат
- **C (Consulted)** - Консультує, надає інформацію
- **I (Informed)** - Інформований про хід виконання

---

## Детальний опис відповідальності

### Визначення бізнес-цілей та критеріїв успіху
- **Accountable:** Спонсор - затверджує фінансові та стратегічні цілі
- **Responsible:** Власник продукту - активно формулює бізнес-цілі
- **Consulted:** Бізнес-аналітик, кінцеві користувачі, експерти безпеки та домену
- **Informed:** Project Manager

### Збір вимог користувача
- **Accountable:** Project Manager - відповідає за повноту процесу
- **Responsible:** Кінцеві користувачі, Бізнес-аналітик
- **Consulted:** Власник продукту, Експерти безпеки

### Аналіз вимог
- **Accountable:** Project Manager
- **Responsible:** Бізнес-аналітик, Системний архітектор
- **Consulted:** Всі зацікавлені сторони

### Проектування архітектури
- **Accountable:** Project Manager
- **Responsible:** Системний архітектор
- **Consulted:** Розробники, Власник продукту, Експерти домену

### Розробка та тестування
- **Accountable:** Project Manager
- **Responsible:** Розробники (розробка), Тестувальники (тестування)
- **Consulted:** Системний архітектор, Експерти

## Комунікаційні канали

| Роль | Частота звітування | Формат | Отримувач |
|------|-------------------|---------|-----------|
| Власник продукту | Щоденно | Stand-up | Команда розробки |
| Бізнес-аналітик | Щотижня | Звіт про вимоги | PM, Власник продукту |
| Системний архітектор | Щотижня | Технічний огляд | PM, Розробники |
| Розробники | Щоденно | Stand-up | PM, Власник продукту |
| PM | Щотижня | Статус проекту | Спонсор, Власник продукту |
