# Горизонтальні прототипи (UI Wireframes)

## Система: SafeHeight Monitor

Горизонтальні прототипи інтерфейсу користувача, створені на мові PlantUML з використанням синтаксису `@salt`.

---

## Список прототипів

### 1. Екран реєстрації працівника
**Файл:** `01_worker_registration.puml`

**Опис:** Інтерфейс реєстрації працівника на початку робочої зміни

**Функціонал:**
- Сканування QR-коду для швидкої реєстрації
- Альтернативна автентифікація через PIN-код (6 цифр)
- Інформаційне повідомлення про наступні кроки
- Візуальна індикація областей вводу

**User Stories:** US1.1, US1.2

**Acceptance Criteria:** AC1.1.1, AC1.1.2, AC1.2.1, AC1.2.2

---

### 2. Панель моніторингу для інженера з охорони праці
**Файл:** `02_monitoring_dashboard.puml`

**Опис:** Головна панель моніторингу всіх працівників на будівельному майданчику в реальному часі

**Функціонал:**
- Таблиця зі списком всіх працівників з статусами (Безпечно/Увага/Критично)
- Кольорова індикація рівня ризику (зелений/оранжевий/червоний)
- Інтерактивна карта майданчику з позначенням працівників
- Фільтри за статусом та зоною
- Пошук працівників
- Автооновлення даних кожні 5 секунд
- Статистика внизу панелі

**User Stories:** US2.1, US2.2

**Acceptance Criteria:** AC2.1.1, AC2.1.2, AC2.2.1

---

### 3. Екран попередження на IoT пристрої
**Файл:** `03_iot_warning_screen.puml`

**Опис:** Мобільний інтерфейс попередження про виявлений ризик падіння на IoT пристрої (браслет/каска з екраном)

**Функціонал:**
- Велике яскраве попередження про критичний ризик
- Детальна інформація про виявлені загрози:
  - Небезпечна висота (12.5 метрів)
  - Нестабільне положення (нахил 45°)
  - Близькість до краю (0.5 метра)
- Ймовірність ризику (95%)
- Рекомендації з безпеки
- Дві кнопки для реакції:
  - "Я в безпеці" - підтвердження отримання
  - "Потрібна допомога!" - активація аварійного протоколу
- Індикація про сповіщення відповідальних осіб
- Статус заряду батареї та з'єднання

**User Stories:** US3.1

**Acceptance Criteria:** AC3.1.1, AC3.1.2, AC3.1.3

---

### 4. Екран керування робочими зонами
**Файл:** `04_work_zones_management.puml`

**Опис:** Інтерфейс для бригадира для створення та керування робочими зонами

**Функціонал:**
- Інтерактивна карта будівельного майданчику
- Можливість малювати нові зони на карті
- Таблиця існуючих зон з інформацією:
  - Назва зони
  - Поверхи
  - Кількість працівників
  - Статус зони
  - Дії (редагувати/видалити)
- Візуалізація працівників на карті (кольорові точки)
- Список бригади з призначеними зонами
- Статуси працівників в реальному часі

**User Stories:** US4.1

**Acceptance Criteria:** AC4.1.1, AC4.1.2

---

### 5. Дашборд звітності з безпеки
**Файл:** `05_safety_report.puml`

**Опис:** Аналітична панель для менеджера проекту з звітами та трендами безпеки

**Функціонал:**
- Вибір періоду звітності (день/тиждень/місяць)
- Експорт звітів в PDF та Excel
- Статистика попереджень:
  - Загальна кількість
  - Розподіл за рівнями (Увага/Критично)
  - Середні показники
  - Пікові значення
- Графік трендів попереджень за період
- Топ-5 зон з найбільшим ризиком (таблиця з відсотками)
- Аналіз інцидентів:
  - Типи виявлених ризиків
  - Розподіл за категоріями
  - Ефективність системи (точність AI, хибні спрацювання, час реакції)
- Рекомендації на основі аналізу даних

**User Stories:** US8.1, US8.2

**Acceptance Criteria:** AC8.1.1, AC8.1.2, AC8.2.1, AC8.2.2

## Технічна інформація

### PlantUML Salt Wireframes

Всі прототипи створені з використанням PlantUML синтаксису `@salt` для створення UI wireframes.

**Переваги PlantUML для прототипування:**
- Текстовий формат (легко зберігати в Git)
- Швидке створення та редагування
- Підтримка таблиць, форм, кнопок, іконок
- Експорт в PNG, SVG
- Автоматична генерація з коду

### Як згенерувати зображення з .puml файлів

**Онлайн:**
1. Відкрити http://www.plantuml.com/plantuml/uml/
2. Вставити вміст .puml файлу
3. Натиснути "Submit" для генерації зображення

**Локально (командний рядок):**
```bash
# Встановити PlantUML
# Для Windows: choco install plantuml
# Для macOS: brew install plantuml
# Для Linux: apt-get install plantuml

# Генерувати PNG з .puml файлу
plantuml 01_worker_registration.puml

# Генерувати всі прототипи
plantuml *.puml
```

**VS Code:**
1. Встановити розширення "PlantUML"
2. Відкрити .puml файл
3. Натиснути Alt+D для попереднього перегляду

## Принципи дизайну

### 1. Візуальна ієрархія
- Критична інформація виділена червоним кольором і великим шрифтом
- Попередження виділені оранжевим
- Безпечні стани - зеленим

### 2. Доступність
- Чіткі заголовки та мітки
- Великі кнопки для легкого натискання (особливо на мобільних пристроях)
- Контрастні кольори для читабельності

### 3. Користувацький досвід
- Мінімум кліків для виконання критичних дій
- Підтвердження для небезпечних операцій
- Інформаційні підказки для нових користувачів

### 4. Реальний час
- Автооновлення даних
- Timestamp для актуальності інформації
- Індикатори з'єднання та синхронізації
